## 一、先搞清楚：IP 被墙和其他问题有什么区别？

VPS 连不上，大概有这几种可能：

- VPS 没有正常开机，或者流量用完被暂停了
- VPS 本身开机了，但 SSH 服务没起来
- SSH 端口被封（IP 本身能 ping 通，但端口访问不了）
- IP 地址在国内被 GFW 屏蔽（也就是"被墙"）
- IP 被搬瓦工官方系统标记封禁（滥用等原因）

这几种情况的症状非常相似——都是"连不上"。但解决方法完全不同，所以检测的第一步是**确认到底属于哪种情况**，别一上来就付钱换 IP，很可能是冤枉钱。

## 二、搬瓦工 IP 被墙检测方法（2026 最新整理）

目前常用的检测手段可以分为三类，建议按顺序来，不要跳步骤。

### 方法一：搬瓦工官方 KiwiVM 黑名单检测工具

这是最权威的检测入口，直接判断 IP 是否被搬瓦工系统标记。

操作步骤很简单：

1. 登录搬瓦工账户后台，进入 Services → My Services；
2. 找到对应的 VPS 实例，点击旁边的齿轮图标，选择 Open KiwiVM；
3. 进入 KiwiVM 控制面板后，手动修改浏览器地址栏末尾：把 `main.php` 替换为 `main-exec.php?mode=blacklistcheck`，回车；
4. 点击 **Test Main IP** 开始检测，等待结果。

这个工具会直接给出 IP 是否被标记为黑名单的结论。如果显示"IP is blocked"，就是官方封禁或国内被墙了。

**注意**：这个检测结果是目前最权威的，建议以此结果为准。

---

### 方法二：port.ping.pe 端口多节点检测（强烈推荐）

有一种情况很让人头大：IP 可以 ping 通，但就是连不上 SSH。这很可能是**端口被封而不是 IP 被封**。

这时候 `port.ping.pe` 这个工具就派上用场了。它可以同时从全球多个节点（包括中国大陆节点）测试你的 IP 某个端口是否可达。

使用方法：打开 `port.ping.pe`，输入格式为 `你的IP:端口号`（例如 `123.45.67.89:28536`），点击 Go 开始测试。

结果解读如下：

- **境外节点绿色（successful）+ 大陆节点红色（failed）**：说明 IP 或端口在国内被封了。先试着改个 SSH 端口，如果换端口也一样，那基本就是 IP 被墙了；
- **大陆境外都是 successful**：端口没问题，连不上可能是其他原因；
- **大陆境外都是 failed**：端口本身没开，检查 VPS 是否正常运行。

---

### 方法三：ping.pe 或 ping.chinaz.com ICMP 检测

这个相对简单，就是向你的 IP 发 ping，看国内外的 ICMP 响应情况。

- **国内节点超时、国外节点正常**：IP 很可能在国内被墙；
- **国内外都超时**：不一定是 IP 被封，更可能是 VPS 关机了，或者防火墙屏蔽了 ICMP；
- **国内外都正常**：IP 本身没问题，去查端口或其他原因。

**重要提示**：单靠 ping 判断已经不够准确，建议 ping 检测和端口检测配合使用，综合判断。

## 三、检测结果怎么解读——四种场景分析

做完上面的检测，基本能对号入座：

**场景 A：KiwiVM 显示 IP 被封，国内端口也测不通**
→ IP 确认被墙，需要付费换 IP（目前官方换 IP 价格约 $8.79，一次性费用），或者购买新套餐。

**场景 B：KiwiVM 显示正常，但 port.ping.pe 显示国内端口不通**
→ 可能是端口被封（不是 IP）。尝试修改 SSH 端口，参考 KiwiVM 面板里的端口修改功能。

**场景 C：KiwiVM 和 port.ping.pe 都显示正常，但就是连不上**
→ 大概率是本地网络问题，或者 VPS 上的 SSH 服务没有正常启动，试试重启 VPS 或重装系统。

**场景 D：所有检测都超时**
→ 先去 KiwiVM 面板查看 VPS 是否在运行状态，确认不是流量用完被暂停。
